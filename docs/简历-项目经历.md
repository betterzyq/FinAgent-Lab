## 排版版（推荐直接粘贴 Word / 飞书简历）

### A股多智能体投研可视化系统（Agent Trading Floor）　　　　独立开发　　　　2025.XX – 2026.06

**项目描述**  
基于 LangGraph 的 TradingAgents 投研引擎，独立实现 **FastAPI + Vue3** 前后端分离可视化层：将 13 个 Agent 的分析、辩论、交易、风控、PM 决策过程，通过 **SSE 实时事件** 驱动 PixiJS 交易大厅动画与报告面板，用于 A 股多 Agent 协作演示与工程验证（非投资建议）。

**项目源码**：`https://github.com/<your-org>/TradingAgents-Visual-Workspace`（请替换）

**技术栈**：LangGraph · FastAPI · DeepSeek · Vue3 · TypeScript · PixiJS · Pinia · SSE · AkShare

**主要工作**

**零侵入 LangGraph 流式观测层**  
针对 TradingAgents 原生 `propagate()` 仅支持 CLI、无法实时观测多 Agent 协作过程的问题，采用 `graph.stream(updates)` 旁路接入，**零修改**核心引擎与原有 CLI 逻辑。自研 `StreamEventTranslator`，将 LangGraph 节点级增量（工具调用、报告生成、辩论轮次、任务交接等）映射为 **15 种**前后端统一的 SSE 事件协议；终态执行 `init_state` 与 stream delta 合并后再持久化，解决 PM 决策完成后因状态字段缺失导致的 `run_error`，保证长链路 Run 稳定结束。

**长任务异步与 SSE 推送架构**  
针对单次 A 股投研分析耗时可达数分钟、同步 HTTP 易超时的问题，设计「POST 立即返回 `run_id` + 后台 Run + SSE 订阅」架构：LangGraph 同步执行放入 `asyncio.to_thread`，线程内产出的事件经 `call_soon_threadsafe` 注入异步 `EventBus`，按 run_id 向多客户端扇出；单订阅队列容量 **512**，慢消费背压丢弃并告警。SSE 连接建立时先 **Replay** 已持久化事件，支持断线重连；前端关闭页面不影响后台任务继续执行，兼顾演示流畅性与任务完整性。

**事件驱动 PixiJS 多 Agent 交易大厅**  
针对多 Agent 流水线阶段多、用户难以感知协作顺序的问题，基于 PixiJS 搭建 **6 大业务区域**（中央控制台、分析师工位、研究辩论室、交易执行台、风控会议室、PM 办公室），为 **13 个** Agent 配置独立 home 坐标与 **7 种**精灵动作（待机 / 工位工作 / 站立辩论 / 跑步交接等）。实现 `useEventDrivenAnimation` 事件—动画状态机：辩论发言、handoff 交付、报告就绪等事件驱动走位与姿态切换；交付点（`HANDOFF_DELIVERY`）与 Agent 工位分离，避免研究经理、交易员在任务交接后占用他人工位，提升场景语义正确性。

**三档演示模式与 LLM Provider 工程化**  
针对开发调试依赖 LLM Token、真实分析配置成本高的问题，设计三层 Mock：**纯前端 Mock**（无后端）、**后端 Mock**（约 **15s** 脚本推送全链路、**0 Token**）、**真实 DeepSeek 分析**三套路径共用同一事件协议。封装 `llm_config` 与启动期 `bootstrap`：仅配置 `DEEPSEEK_API_KEY` 即可自动识别 Provider 并映射 `deepseek-v4-flash / v4-pro` 默认模型，降低多 Provider 环境下的接入与演示成本。

**全链路投研 UI 与 A 股数据接入**  
基于 Pinia 统一消费 SSE，联动 Agent 状态栏、分角色分析报告、多空/风控辩论面板、事件时间线与 PM 最终决策弹窗（长报告 **可滚动**），形成「看得见、读得懂、可追溯」的演示闭环。后端对接 AkShare 实现 A 股代码/名称搜索（**24h** 缓存），Vite 开发态 `/api` 代理 + TypeScript 类型约束覆盖事件与 Agent 枚举，保证前后端协议一致。
