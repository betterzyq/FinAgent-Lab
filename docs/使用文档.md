# TradingAgents 可视化系统 · 前后端使用文档

> **Agent Trading Floor / A股智能投研交易室**  
> 前后端分离架构：FastAPI 后端 + Vue 3 / PixiJS 前端  
> 本系统仅供研究与演示，**不构成任何投资建议**。

---

## 一、系统概览

```
┌─────────────────────────────────────────────────────────────┐
│  浏览器 · http://127.0.0.1:5173                              │
│  ┌──────────┬──────────────────────────┬──────────────────┐ │
│  │ Agent    │  PixiJS 交易大厅动画      │  报告 / 辩论     │ │

│  └──────────┴──────────────────────────┴──────────────────┘ │

└────────────────────┼────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  FastAPI 后端 · http://127.0.0.1:8000                       │
```

## 二、环境要求

| 组件 | 版本要求 |
|------|----------|
| Python | 3.10+ |
| Node.js | ^20.19 或 >=22.12 |
| 操作系统 | Windows / macOS / Linux |

**真实分析运行**还需：

- 已安装并可导入 `TradingAgents` 包
- `TradingAgents/.env` 中配置 LLM API Key（如 OpenAI、DeepSeek 等）

**Mock 演示**无需 LLM，可直接体验完整动画与事件流。

## 三、安装

在工作区根目录 `d:\TradingAgents-Visual-Workspace` 执行：

### 3.1 安装 TradingAgents 核心

```powershell
cd d:\TradingAgents-Visual-Workspace
pip install -e TradingAgents
```

### 3.2 安装后端依赖

```powershell
pip install -r backend/requirements.txt
```

### 3.3 安装前端依赖

```powershell
cd frontend
npm install
# 若网络受限，可使用本地缓存：
# npm install --cache .npm-cache
```

### 3.4 配置环境变量（真实运行）

编辑 `TradingAgents/.env`，至少配置 **一个 LLM 提供商的 API Key**：

```env
# 使用 DeepSeek（推荐，仅需这一行即可自动识别）
DEEPSEEK_API_KEY=sk-your-key-here

# 可选：显式指定 provider 与模型（不设置也会自动推断）
# TRADINGAGENTS_LLM_PROVIDER=deepseek
# TRADINGAGENTS_QUICK_THINK_LLM=deepseek-v4-flash
# TRADINGAGENTS_DEEP_THINK_LLM=deepseek-v4-pro
```

| 场景 | 所需配置 |
|------|----------|
| **DeepSeek** | 仅 `DEEPSEEK_API_KEY`（后端会自动选用 DeepSeek 及对应模型） |
| **OpenAI** | `OPENAI_API_KEY`，或额外设置 `TRADINGAGENTS_LLM_PROVIDER=openai` |
| **其他厂商** | 对应 Key（如 `GOOGLE_API_KEY`）+ 可选 `TRADINGAGENTS_LLM_PROVIDER` |

后端启动时会自动加载 `TradingAgents/.env`。

## 四、启动服务

### 4.1 启动后端

**在工作区根目录**执行（不要在 `backend/` 子目录内）：

```powershell
cd d:\TradingAgents-Visual-Workspace
uvicorn backend.main:app --reload --host 127.0.0.1 --port 8000
```

| 地址 | 说明 |
|------|------|
| http://127.0.0.1:8000/docs | Swagger API 文档 |
| http://127.0.0.1:8000/api/health | 健康检查（含已识别的 LLM provider） |

健康检查示例（DeepSeek 用户应看到 `llm_provider: deepseek`）：

```json
{
  "status": "ok",
  "llm_provider": "deepseek",
  "quick_think_llm": "deepseek-v4-flash",
  "deep_think_llm": "deepseek-v4-pro",
  "deepseek_key_set": "True",
  "openai_key_set": "False"
}
```

若 `llm_provider` 仍为 `openai` 且 `deepseek_key_set` 为 `False`，说明 `.env` 未加载——请 **完全重启** 后端（Ctrl+C 后重新运行 uvicorn）。

### 4.2 启动前端

```powershell
cd d:\TradingAgents-Visual-Workspace\frontend
npm run dev
```

| 地址 | 说明 |
|------|------|
| http://127.0.0.1:5173 | 可视化界面 |

前端开发服务器已将 `/api` 代理到 `http://127.0.0.1:8000`（见 `frontend/vite.config.js`）。

### 4.3 生产构建（可选）

```powershell
cd frontend
npm run typecheck
npm run build
npm run preview   # 本地预览 dist
```

构建产物在 `frontend/dist/`。部署时需自行配置反向代理，将 `/api` 转发到后端。

## 五、前端使用指南

### 5.1 界面布局

| 区域 | 功能 |
|------|------|
| **顶部控制栏** | 股票代码、名称、日期、分析深度、启动/重置、Mock 开关 |
| **左侧 Agent 状态栏** | 13 个 Agent 实时状态、选中查看详情 |
| **中央 PixiJS 场景** | 交易大厅动画：工位、辩论室、风控室、PM 办公室 |
| **右侧报告面板** | 各 Agent 生成的分析报告 |
| **右侧辩论面板** | 研究辩论、风控辩论消息 |
| **底部时间线** | 按时间顺序展示全部 SSE 事件 |

### 5.2 基本操作流程

1. 打开 http://127.0.0.1:5173
2. 确认右上角连接状态为 **「已连接」**（需后端运行）
3. 输入 **股票代码**（如 `600519`），名称会自动联想填充
4. 选择 **分析日期**、**分析深度**、辩论轮数
5. 选择运行模式（见下节）
6. 点击 **「启动分析」**
7. 观察中央场景 Agent 动画、右侧报告与时间线
8. 分析结束后查看 **PM 最终决策卡片**
9. 点击 **「重置」** 清空状态，开始新一轮

### 5.3 三种运行模式

| 模式 | 勾选方式 | 说明 |
|------|----------|------|
| **纯前端 Mock** | 勾选「纯前端 Mock」 | 完全不依赖后端，本地模拟 SSE 事件，适合调试动画 |
| **后端 Mock** | 勾选「后端 Mock」 | 调用 `POST /api/runs?mock=true`，约 10 秒完整事件链 |
| **真实分析** | 两个 Mock 均不勾选 | 调用 TradingAgents 完整 LangGraph 流程，需 LLM Key |

推荐初次体验顺序：**纯前端 Mock** → **后端 Mock** → **真实分析**。

### 5.4 前端环境变量

在 `frontend/.env` 或 `frontend/.env.local` 中：

```env
# 留空 = 使用相对路径，走 Vite 代理（开发推荐）
VITE_API_BASE_URL=

# 生产部署示例：指向独立后端域名
# VITE_API_BASE_URL=https://api.example.com
```

## 六、后端 API 参考

### 6.1 健康检查

```http
GET /api/health
```

响应：`{"status":"ok"}`

### 6.2 创建分析任务

```http
POST /api/runs
POST /api/runs?mock=true    # Mock 模式
Content-Type: application/json
```

请求体：

```json
{
  "symbol": "600519",
  "company_name": "贵州茅台",
  "market": "A股",
  "trade_date": "2026-06-17",
  "analysis_depth": "standard",
  "debate_rounds": 2,
  "risk_rounds": 2
}
```

| 字段 | 说明 |
|------|------|
| `symbol` | A 股代码，如 `600519` 或 `600519.SS` |
| `company_name` | 公司名称（可选，用于 UI 展示） |
| `trade_date` | 分析基准日，`YYYY-MM-DD` |
| `analysis_depth` | `quick` / `standard` / `deep` |
| `debate_rounds` | 研究辩论轮数，1–5 |
| `risk_rounds` | 风控辩论轮数，1–5 |

响应：

```json
{
  "run_id": "uuid-string",
  "status": "running",
  "message": "Analysis run started"
}
```

### 6.3 订阅 SSE 事件流

```http
GET /api/runs/{run_id}/events
Accept: text/event-stream
```

- 先回放已持久化事件，再实时推送
- 每 30 秒发送 `heartbeat` 保活
- `run_finished` 或 `run_error` 后流结束

前端使用 `EventSource` 订阅，见 `frontend/src/api/tradingApi.ts`。

### 6.4 查询任务状态

```http
GET /api/runs/{run_id}
```

返回运行状态、当前阶段、最终决策等。

### 6.5 查询完整时间线

```http
GET /api/runs/{run_id}/timeline
```

返回该任务全部历史事件（JSON 数组）。

### 6.6 Agent 列表

```http
GET /api/agents
```

返回 13 个 Agent 的静态配置（id、名称、团队等）。

### 6.7 股票搜索

```http
GET /api/symbols/search?keyword=600519&limit=20
```

按代码前缀或名称模糊搜索 A 股；AkShare 不可用时返回内置 Mock 列表。

## 七、SSE 事件协议

前端与后端通过以下事件类型驱动动画与 UI，**请勿随意修改事件名**。

| 事件类型 | 说明 | 前端行为 |
|----------|------|----------|
| `run_started` | 任务启动 | 大屏显示股票信息，协调官就位 |
| `agent_started` | Agent 开始工作 | 对应小马移动到工位、working 状态 |
| `agent_finished` | Agent 阶段完成 | 状态变为 finished |
| `tool_called` | 调用工具 | 保持 working |
| `report_ready` | 报告生成 | 右侧报告面板追加内容 |
| `handoff` | 任务交接 | 跑步动画 + 临时轨迹 |
| `debate_message` | 研究辩论发言 | 辩论室动画 + 右侧面板 |
| `debate_finished` | 研究辩论结束 | 研究经理总结 |
| `trader_proposal_ready` | 交易提案 | 交易员 → 风控室 |
| `risk_debate_message` | 风控辩论发言 | 风控室动画 |
| `risk_debate_finished` | 风控辩论结束 | — |
| `pm_decision_ready` | PM 最终决策 | 决策卡片 + 大屏更新 |
| `run_finished` | 全部完成 | 全员回 idle/finished |
| `run_error` | 运行出错 | 错误状态 + 时间线标记 |

事件 JSON 结构（核心字段）：

```json
{
  "event_id": "uuid",
  "run_id": "uuid",
  "timestamp": "2026-06-17T12:00:00Z",
  "event_type": "agent_started",
  "agent_id": "market_analyst",
  "agent_name": "行情技术分析师",
  "message": "开始分析",
  "payload": { }
}
```

> 后端 `news_analyst` 会在前端映射为 `news_policy_analyst`。

## 十、常见问题

### Q1：前端显示「未连接」

- 确认后端已启动：`uvicorn backend.main:app --reload --host 127.0.0.1 --port 8000`
- 访问 http://127.0.0.1:8000/api/health 应返回 `ok` 且 `llm_provider: deepseek`
- 或勾选「纯前端 Mock」跳过后端

### Q1b：health 正常但分析仍报 OpenAI Key 错误

**常见原因：8000 端口被两个旧后端进程占用。** 前端代理连的是 `127.0.0.1:8000`，可能连到旧进程。

PowerShell 检查：

```powershell
netstat -ano | findstr ":8000"
```

若看到两个 `LISTENING`（例如 `0.0.0.0:8000` 和 `127.0.0.1:8000`），结束多余进程后重启：

```powershell
# 先 Ctrl+C 停掉当前 uvicorn，再查 PID
netstat -ano | findstr ":8000"
taskkill /PID <多余进程的PID> /F

cd D:\TradingAgents-Visual-Workspace
uvicorn backend.main:app --reload --host 127.0.0.1 --port 8000
```

确认 health 返回 `"api_version":"0.1.1"` 且 `"llm_provider":"deepseek"`。

### Q2：真实分析报错 `API key for provider 'openai' is not set`

- 默认 provider 为 OpenAI；若你使用的是 **DeepSeek**，只需在 `TradingAgents/.env` 中设置 `DEEPSEEK_API_KEY`，后端会自动识别（无需 OpenAI Key）
- 也可显式指定：`TRADINGAGENTS_LLM_PROVIDER=deepseek`
- 修改 `.env` 后需 **重启后端**（`uvicorn`）
- 查看后端终端日志，应出现 `LLM provider auto-detected as 'deepseek'`

### Q3：CORS 跨域错误

后端默认允许 `5173`、`8090` 端口。若使用其他端口，修改 `backend/main.py` 中 `allow_origins`。

### Q4：`pip install -e TradingAgents` 失败

- 确认 Python 版本 >= 3.10
- 在 TradingAgents 目录查看其 README 安装说明

### Q5：`npm run build` 失败

```powershell
cd frontend
npm run typecheck   # 查看 TypeScript 错误
npm run build
```

### Q6：前端断开 SSE 后任务是否继续？

**会继续。** 后端任务在后台执行，事件写入内存 `RunStore`。重新打开页面后可通过 `GET /api/runs/{run_id}/timeline` 获取历史事件。
